### **Feature 3: Advanced Deduplication** ðŸŽ¯

**What Users Get:**  
Automatically merge duplicate entities so "AI", "A.I.", and "Artificial Intelligence" become ONE clean node.

**How It Works - 3-Tier Scoring:**

**Tier 1: Fuzzy String Matching (30% weight)**
- Catches typos: "Googel" â†’ "Google"
- Catches variations: "Inc." â†’ "Incorporated"
- Levenshtein distance algorithm
- Fast initial pass

**Tier 2: Semantic Similarity (30% weight)**
- Compares meaning via embeddings (384-dimensional vectors)
- Catches synonyms: "ML" â†” "Machine Learning"
- Catches abbreviations: "AI" â†” "Artificial Intelligence"
- Cosine similarity threshold: 0.85

**Tier 3: LLM Validation (40% weight)**
- Gemini API validates uncertain cases
- Checks context: "Is 'Bank' (financial) same as 'Bank' (river)?" â†’ No
- Checks abbreviations: "Is 'WHO' the same as 'World Health Organization'?" â†’ Yes
- Only called for medium-confidence cases (saves API costs)

**Merge Decision Logic:**
- **Total Confidence >90%:** Auto-merge, notify user
- **Total Confidence 70-90%:** Suggest merge, user confirms
- **Total Confidence <70%:** Keep separate

**Why It Matters:**
- Clean graphs without duplicate clutter
- "AI" (10 mentions) + "Artificial Intelligence" (15 mentions) = One node with 25 mentions
- Preserves aliases so users see all variations
- 85-90% deduplication accuracy (vs 60% for competitors)

**Competitive Edge:**
- InfraNodus: String-matching only (60% accuracy, misses synonyms)
- Neo4j: Requires manual MERGE queries (technical, error-prone)
- Obsidian: No deduplication (massive duplicates)
- Notion: No deduplication

**User Experience:**
- Deduplication log shows what was merged:
  - "Merged 'AI' â† 'A.I.', 'Artificial Intelligence'"
  - "Merged 'Google' â† 'Google Inc.', 'Google LLC'"
- Click any node â†’ see aliases:
  - Main name: "Artificial Intelligence"
  - Also known as: "AI", "A.I."
- "Undo merge" button if user disagrees
- "Suggest more merges" button for power users
- Manual merge option: drag one entity onto another

### **Feature 4: Relationship Detection** ðŸ”—

**What Users Get:**  
Automatically find meaningful connections between entities with confidence scoring.

**How It Works - Multi-Layer Detection:**

**Layer 1: Co-occurrence**
- Entities appear in same sentence â†’ connected
- Frequency tracking: mentioned together 10Ã— â†’ strong connection
- Example: "AI and Machine Learning" appear together 15 times â†’ thick edge

**Layer 2: Semantic Similarity**
- Embeddings are close â†’ conceptually related
- Even if not mentioned together, show connection
- Example: "Neural Networks" and "Deep Learning" (semantically similar) â†’ dashed edge

**Layer 3: LLM Classification (Feature 20 - Tier 1)**
- Gemini classifies relationship type based on context
- Extracts specific relationship from sentences
- Types: is_a, part_of, causes, uses, created_by, related_to
- Example: "Deep Learning is a subset of Machine Learning" â†’ classified as "is_a"

**Relationship Strength:**
- Based on: frequency + semantic similarity + context
- **Thick solid line:** Strong connection (10+ co-occurrences)
- **Thin solid line:** Weak connection (1-5 co-occurrences)
- **Dashed line:** Semantic similarity only (not co-occurring)

**Confidence Filtering:**
- Only show high-confidence relationships (>70% by default)
- Reduces noise (no random connections from headers/footers)

**Why It Matters:**
- Shows HOW concepts connect, not just THAT they exist
- Reduces hairball graphs (only meaningful connections)
- Context-aware (distinguishes "Apple makes iPhones" from "Apple is a fruit")

**Competitive Edge:**
- Obsidian: Shows ALL links (extremely noisy)
- InfraNodus: Shows too many edges (hairball problem)
- Neo4j: Manual relationship creation
- Notion: No relationships at all

**User Experience:**

  - "Mentioned together 12 times in 4 documents"
  - Relationship type: "uses" or "is_a" or "related_to"
  - Confidence: 85%
- Click edge â†’ shows example sentences:
  - "Google uses TensorFlow for machine learning."
